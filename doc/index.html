OOP Concept Cross-Reference

Concept Where to Find It
Abstraction Vehicle (abstract class), HeavyVehicle (abstract class), all four interfaces
Encapsulation Every model class — private fields + validated setters (see Driver, Vehicle, components)
Hierarchy (Inheritance) Vehicle → Car/Motorcycle, Vehicle → HeavyVehicle → Truck/Bus
Polymorphism calculateOperatingCost() in each subclass, Fleet.calculateTotalOperatingCost(), Fleet.generateFleetReports()
Composition Vehicle owns Engine, Transmission, BreakingSystem; MainWindow owns panels + service
Aggregation Fleet ◇→ Vehicle/Driver; panels ◇→ FleetService; FleetService ◇→ Fleet
Upcasting Any subclass stored as Vehicle (e.g. Fleet.addVehicle(new Car(...)))
Downcasting Fleet.getVehicleSpecificDetails() and Fleet.getVehiclesRequiringCommercialLicense() — instanceof + explicit cast
Modularity Package separation: app → gui → service → model; each panel is a self-contained module
